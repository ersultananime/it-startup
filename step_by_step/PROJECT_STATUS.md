# Отчет о статусе проекта Step by Step

## ✅ Реализовано: Что уже есть в коде

*   **Окружение и Git**:
    *   Есть файл `.gitignore` (подразумевается, раз есть папки `.mypy_cache`, `.pytest_cache`, `__pycache__`).
    *   Есть базовая структура сервера FastAPI (`app/`, `tests/`), шаблоны (`templates/`) и статика (`static/`).
*   **Реализация БД (SQL)**:
    *   Логика подключения к БД (SQLite) настроена (`database.py`, `app/database.py`).
    *   Создание таблиц происходит автоматически при запуске (`Base.metadata.create_all(bind=engine)` в `main.py` и `app/main.py`).
*   **Основы API**:
    *   Настроен веб-сервер на **FastAPI**.
    *   Присутствуют базовые эндпоинты (например, `/` возвращает HTML).
*   **CRUD Операции**:
    *   Часть 1: Реализованы GET запросы (получение профиля `/users/{id}`, цели `/goals/{id}`) и POST запросы (создание пользователя `/users/`, цели `/goals/`, лога активности `/log-activity`, регистрации `/api/register`, логина `/api/login`, апдейт веса `/update_weight`).
*   **Обработка ошибок**:
    *   Реализована обработка ошибок с возвратом 404 (объект не найден) и 400/401/403/409 статусов (например, в `app/routers/users.py`, `app/routers/goals.py`, `app/routers/activity.py` и `main.py`).
*   **Валидация и Безопасность**:
    *   Частичная защита от SQL-инъекций за счет использования ORM (SQLAlchemy).
    *   Валидация данных с помощью Pydantic схем (`app/schemas.py`).
*   **Аутентификация**:
    *   Есть таблица пользователей (`users`).
    *   Реализована базовая система регистрации и логина с использованием сессионных куки (`session_id`).
*   **Сложные запросы**:
    *   Используются связи между таблицами (например, получение логов для конкретного пользователя и цели в `app/routers/activity.py`, связи `relationship` в моделях).
*   **Интеграция и Документация**:
    *   Генерируется Swagger/OpenAPI документация FastAPI (доступна по `/docs`).
    *   Есть файл `README.md` с инструкцией по запуску и описанием архитектуры.
*   **Деплой и Тесты**:
    *   Поддержка переменных окружения через файл `.env` (и `.env.example`).
    *   Наличие базовых тестов функционала (`tests/test_api.py`).

## ❌ Отсутствует: Чего не хватает согласно списку выше

*   **Окружение и Git**:
    *   Нет четко выделенных папок `data` и `docs`.
    *   Отсутствуют коммиты в Git репозитории (репозиторий пуст или не инициализирован для ветки master - `fatal: your current branch 'master' does not have any commits yet`).
*   **Проектирование БД**:
    *   Отсутствует ER-диаграмма или отдельный файл описания схемы (хотя схема описана в `models.py`).
    *   Спорный момент с нормализацией: В MVP `main.py` пароли и данные хранятся в одной таблице. Данные профиля `users` (рост, вес) можно было бы вынести в `profiles` таблицу.
*   **Реализация БД (SQL)**:
    *   Нет явных DDL-скриптов (.sql файлы), так как SQLAlchemy генерирует таблицы "на лету".
    *   Скрипты для заполнения данными (Seeding) отсутствуют (заполнение происходит динамически через API `POST` или при регистрации). В тестах используется динамическое создание (`_create_user`).
*   **CRUD Операции**:
    *   Часть 2: **Отсутствуют PUT/PATCH и DELETE запросы**. Представлены только GET и POST.
*   **Валидация и Безопасность**:
    *   **Отсутствует хеширование паролей**. В `main.py` прямо сказано: `# Plain text for MVP as per plan` (строка 163). Пароли хранятся и проверяются в открытом виде.
*   **Интеграция и Документация**:
    *   Нет экспортированной коллекции Postman (есть Swagger).

## 💡 Рекомендации

1.  **Коммиты Git**: Инициализировать репозиторий и сделать первый коммит (`git init`, `git add .`, `git commit -m "Initial commit"`).
2.  **Структура папок**: Создать папки `data` (для БД, если это SQLite файлы) и `docs` (туда поместить ER-диаграмму и другие документы).
3.  **Безопасность (Пароли)**: Добавить библиотеку `passlib` (например, `passlib[bcrypt]`), хешировать пароль при регистрации (`app/routers/users.py` и `main.py`) и проверять хеш при логине.
4.  **CRUD (PUT/DELETE)**:
    *   Добавить `/users/{id}` (PUT/PATCH) для обновления данных пользователя (изменение роста, веса, цели).
    *   Добавить `/users/{id}` (DELETE) для удаления профиля.
    *   Добавить `/goals/{id}` (PUT/PATCH/DELETE) для управления целями.
    *   Обновить `app/routers/users.py` и `app/routers/goals.py`.
5.  **Seeding БД**: Создать отдельный скрипт `seed.py`, который будет подключаться к базе (через `database.py`) и создавать несколько пользователей по умолчанию (с хешированными паролями) и тестовые данные (логи, цели).
6.  **ER Диаграмма**: Нарисовать схему и добавить в Markdown (или PNG) в `/docs/schema.md`.
